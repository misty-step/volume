## Architecture Critique Summary

Three parallel investigation tracks:

- **Track A** (reference architectures): ChatGPT memory (4-layer, flat text), Mastra (observer/reflector), Fitbit AI Coach (proactive suggest-and-explain), Fitbod (curated taxonomy)
- **Track B** (domain audits): LLM infra (zero observability, no evals, two OpenRouter clients), context engineering (naive sliding window, no user context injection), harness (52% coverage vs 70% documented)
- **Track C** (thinktank + Codex): 6/10 architecture rating. "Prototype-grade, not strong agent-product architecture." Biggest risk: no durable conversation state. Kill: API route pass-through. Double down: block schema.

Codex: "The biggest problem is not 'too many tool files.' It's that the system is pretending to be chat-first without giving chat state a real home."

## Research Findings (Preserved)

### `@convex-dev/agent` (v0.3.2)

- Persistent threads, delta streaming (websocket), vector+text search, contextHandler
- AI SDK 6 supported since v0.6.0
- Pre-1.0, 36 open issues, data-corruption bugs (#200), infinite tool loop (#167)
- Our custom block schema (12 types) doesn't map to UIMessage format
- Tool rewrite required (ConvexHttpClient → ctx.runQuery) — significant migration cost
- **Verdict: Wait for 1.0. Build conversation persistence on plain Convex.**

### json-render (v0.14.0)

- 92k/week downloads, actively maintained (released yesterday)
- `defineCatalog()` + `defineRegistry()` pattern
- AI SDK 6 inline mode: `pipeJsonRender` + `useJsonRenderMessage`
- 36 shadcn components available, custom components fully supported
- Our 12 blocks map cleanly to catalog entries
- Key tension: tools return data (deterministic), model generates UI spec (non-deterministic)
- Known infinite re-render bug (#53)
- **Verdict: Proceed with migration. Design: tools return data, model generates spec.**

### Memory architecture

- ChatGPT: 4-layer (session, saved facts ~33, conversation summaries, current session). Flat text, no vector DB.
- Mastra: Observer (compress at 30k tokens) + Reflector (GC at 40k tokens). 94.87% LongMemEval.
- Mem0: Extraction → Update (ADD/UPDATE/DELETE). 26% accuracy boost, 90% token savings.
- **Verdict: Hybrid — saved facts (ChatGPT) + observation log (Mastra). No vector DB at current scale.**

### Exercise identity

- No major fitness app does semantic entity resolution at NLP level — all curated taxonomies
- Agent-forward: LLM already in the loop for the coach turn, use it for resolution
- Embeddings considered but deferred (overkill for ~500 exercises, adds complexity)
- **Verdict: LLM-as-resolver (already in the loop) + curated canonical list + alias table**
