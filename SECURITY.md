# Security Policy

## Supported Branch

Security fixes should target `master`.

## Reporting a Vulnerability

Do not open a public issue for credential leaks, auth bypasses, billing flaws,
data exposure, or anything that could put user data at risk.

Report vulnerabilities privately through GitHub's "Report a vulnerability"
flow for this repository. If that flow is unavailable, contact
[@phrazzld](https://github.com/phrazzld) privately on GitHub with:

- A clear description of the issue
- Reproduction steps or a proof of concept
- Impacted routes, pages, or backend functions
- Any suggested mitigation

Please redact secrets, tokens, and user data from screenshots or logs.

## Handling Expectations

- We will acknowledge receipt and evaluate impact.
- We may ask for follow-up details or a minimal reproduction.
- Please avoid public disclosure until a fix or mitigation is in place.
