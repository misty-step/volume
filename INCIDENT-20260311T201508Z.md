# Incident: Mobile Production Spinner On Authenticated App Routes

**Date:** 2026-03-11
**Severity:** High
**Status:** Fixed in code, local verification in progress

## Timeline (UTC)

| Time                 | Event                                                                                                                 |
| -------------------- | --------------------------------------------------------------------------------------------------------------------- |

## Evidence

- [`src/components/subscription/paywall-gate.tsx`](src/components/subscription/paywall-gate.tsx) queried `api.users.getSubscriptionStatus` immediately, before client auth readiness was proven.
- [`convex/users.ts`](convex/users.ts) returned `null` for both unauthenticated access and missing user records.
- The gate treated both `undefined` and `null` as spinner states, and interpreted `null` as “create the user,” even when Convex auth was not ready yet.
- The gate only used `console.warn` / `console.error`, so the failure mode could remain invisible to Sentry/PostHog unless another layer threw.

## Root Cause

`PaywallGate` used one ambiguous signal for three different conditions:

1. Clerk not loaded yet
2. Convex auth not ready or not authenticated yet
3. Authenticated user exists in Clerk but no user record exists in Convex

Because `getSubscriptionStatus` returned `null` for both unauthenticated and missing-user states, the client gate could not tell whether it should wait, create a user, or fail. On slower mobile auth bootstrap, that ambiguity degraded into a fullscreen spinner with no terminal error UI.

## Why Sentry/PostHog Did Not Clearly Show It

- No `trackEvent(...)` calls existed in `PaywallGate`.
- No `reportError(...)` calls existed in `PaywallGate`.
- Global error boundaries only report thrown route errors, but this flow mostly spun and logged with `console.*`.
- Direct dashboard inspection from this workspace was blocked by missing local credentials/config for Convex/Vercel CLI, so the investigation relied on code-path evidence plus public health checks.

## Fix

- Gate `getSubscriptionStatus` behind explicit auth readiness (`Clerk + Convex`).
- Treat “user record missing” as a separate post-auth state instead of conflating it with auth bootstrap.
- Replace endless spinner behavior with a timeout-backed recovery UI.
- Emit Sentry/PostHog telemetry for delayed bootstrap, failed user bootstrap, and checkout verification timeout.

## Preventive Work

- Added focused unit coverage for auth readiness and user-bootstrap failure paths.
- Added a mobile-oriented Playwright regression around authenticated reload.
- Added stable `data-testid` hooks for spinner/error states to support future E2E coverage.
