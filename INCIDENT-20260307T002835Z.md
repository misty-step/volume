## Incident

- **Timestamp (UTC)**: 2026-03-07T00:28:35Z
- **Symptom**: Production coach request like `30 squats` returned `The AI runtime is unavailable right now. Please try again shortly.`
- **Impact**: `/api/coach` unusable in production for authenticated users

## Timeline

- `2026-03-07T00:28:58Z` `GET https://volume.fitness/api/health` returned `503` with `checks.coachRuntime.status=fail` and `missing=["OPENROUTER_API_KEY"]`
- `2026-03-07T00:29Z` `./scripts/verify-env.sh --prod-only` passed because it only validated Convex production env vars
- `2026-03-07T00:29Z` `bunx convex env list --prod` confirmed Convex production had `OPENROUTER_API_KEY`
- `2026-03-07T00:29Z` `vercel env ls` confirmed Vercel production did **not** have `OPENROUTER_API_KEY`
- `2026-03-07T00:31Z` `vercel inspect volume.fitness --logs` showed current prod build succeeded without validating the missing Vercel runtime var
- `2026-03-07T00:40Z` after restoring Vercel envs and redeploying, `GET https://volume.fitness/api/health` returned `200` with `checks.coachRuntime.status=pass`
- `2026-03-07T00:40Z` final Vercel redeploy `dpl_EPwMdDC3M5UF7xQyH6B1D5EzqJsp` reached `Ready`

## Evidence

### Health endpoint

- `/api/health` computes coach health from `process.env.OPENROUTER_API_KEY?.trim()`
- Current production response reported coach runtime missing

### Code path

- [`src/components/coach/useCoachChat.ts`](./src/components/coach/useCoachChat.ts) posts coach turns to [`src/app/api/coach/route.ts`](./src/app/api/coach/route.ts)
- [`src/app/api/coach/route.ts`](./src/app/api/coach/route.ts) calls `getCoachRuntime()`
- [`src/lib/coach/server/runtime.ts`](./src/lib/coach/server/runtime.ts) returns `null` when `OPENROUTER_API_KEY` is missing or blank
- [`src/app/api/coach/route.ts`](./src/app/api/coach/route.ts) converts `runtime === null` into `buildRuntimeUnavailableResponse()`
- [`src/lib/coach/server/blocks.ts`](./src/lib/coach/server/blocks.ts) owns the exact user-facing error message

### Deployment/config split

- Convex production env validation lived in [`scripts/verify-env.sh`](./scripts/verify-env.sh)
- Vercel production env validation was absent
- Deployment docs listed `OPENROUTER_API_KEY` only under Convex, not Vercel

### Secondary finding

- `vercel inspect volume.fitness --logs` showed Sentry build integration errors caused by malformed Vercel env values for Sentry project config
- This did **not** cause the coach outage, but it proves the Vercel env surface lacked validation and observability
- Final redeploy log showed `Successfully uploaded source maps to Sentry` after normalizing the malformed Vercel env values

## Hypotheses

1. **ROOT**: Vercel production is missing `OPENROUTER_API_KEY`, so Next.js `/api/coach` disables the runtime
2. **SYMPTOM**: Planner/tool execution is broken after runtime initialization
3. **SYMPTOM**: Paywall/subscription gating blocks coach

## Actions

- Pulled production health response
- Compared Convex prod env vs Vercel prod env
- Traced `/today` UI -> `/api/coach` -> `getCoachRuntime()` -> `runtime unavailable`
- Checked Sentry CLI for matching issues and found none
- Checked Vercel deployment logs for current prod deployment
- Restored `OPENROUTER_API_KEY` to Vercel production
- Redeployed current production deployment
- Hardened `scripts/verify-env.sh` to validate Vercel production envs too

## Root Cause

Primary root cause: **Vercel production did not have `OPENROUTER_API_KEY`**, while the coach runtime runs inside Next.js route handlers on Vercel, not inside Convex. The existing verification script only checked Convex production env vars, so deploy safety passed even though the runtime-critical Vercel env was missing.

## Fix

- Immediate: add `OPENROUTER_API_KEY` to Vercel production and redeploy
- Strategic: make production env verification a single module that validates both Convex and Vercel env surfaces, including blank/whitespace-corrupted values

## Verification

- `curl -sSL https://volume.fitness/api/health` returned:
  - `status=pass`
  - `checks.coachRuntime.status=pass`
- `./scripts/verify-env.sh --prod-only` now passes and validates both Convex and Vercel production env surfaces
- Final Vercel build logs showed Sentry source-map upload succeeding instead of failing on malformed `SENTRY_PROJECT`
