## Research Findings

- AI SDK docs reinforce the chosen boundary: tools are model-controlled, and loop control belongs in `stopWhen` / agent orchestration rather than ad hoc route logic.
- Mastra docs reinforce memory as a layered concern (working memory + semantic recall + thread persistence), which argues for a clean adapter boundary instead of stuffing more state into the client hook.
- `json-render` is built around a constrained catalog and streamed JSON patches, which fits the repo's own design docs and argues against continuing ad hoc semantic ladders and inline renderer coupling.
- Modern agent operations also need explicit evals, trace review, and canonical model/routing governance. The backlog was missing those until #416, #386, and #339 were promoted into the roadmap.
- The repo has no obvious sibling-repo prior art inside `misty-step` for this exact coach/generative-UI stack, so the cleanest leverage is from internal docs and issue consolidation, not code reuse.

## Retro Patterns Applied

- Avoided broad “ship the whole pivot” scoping because transport/render work has already expanded unexpectedly
- Treated docs/package mismatch risk as a first-class concern when deciding what to keep
- Elevated issue quality to a delivery concern because screenshot-only tickets are not agent-executable
- Distinguished agent product work from agent operations work so the roadmap now covers both UX contract and LLM reliability discipline
